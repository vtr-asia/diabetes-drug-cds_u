臺灣50優良SMART App 徵案 - 第二階段測試啟動頁

請選擇下方測試模式，點擊「立即測試」即可使用官方提供的 Client ID 與 Client Secret 進行授權流程。

重要說明：
此組 Client ID 及 Client Secret 僅供內部測試使用，不供未來產品化使用。
若您需要提供測試資料，請以 FHIR Bundle JSON 格式提供，並請勿提供真實病患資料。

選擇測試模式

1. EHR Launch（從 EHR 內啟動）

Launch URL、Redirect URI、Scopes

程式碼 
FHIR.oauth2.authorize({
  clientId: "cc344727-6f90-496c-94fd-c7829aa9a51d",
  clientSecret: "79f04b56b33491716c0880af72cdef7d3f0629111421cedd18353651cd313d9e",
  scope: "launch openid fhirUser patient/*.read",
  redirectUri: window.location.origin + "/index.html"
});

2. Standalone Launch（獨立啟動）

Redirect URI、Scopes（ISS 設為官方 Launcher）

程式碼 
FHIR.oauth2.authorize({
  clientId: "cc344727-6f90-496c-94fd-c7829aa9a51d",
  clientSecret: "79f04b56b33491716c0880af72cdef7d3f0629111421cedd18353651cd313d9e",
  scope: "openid fhirUser patient/*.read",
  iss: "官方提供的 ISS URL",   // 請替換為官方 Launcher ISS
  redirectUri: window.location.origin + "/index.html"
});

3. Backend Service（後端服務）

Scopes、JWKS 或 JWKS URL

（此模式需在 Node.js 後端執行，clientSecret 不可暴露於前端）

測試資料準備

若有特定測試資料，請以 FHIR Bundle JSON 格式提供，並壓縮成 ZIP 或 7Z 檔後提供給官方。

完成準備後，請提供至少 3 個測試時段（每個團隊約 30 分鐘）。
